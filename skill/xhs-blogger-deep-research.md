# 小红书博主深度研究分析框架

> GitHub 仓库: `https://github.com/fjbg1262-byte/xhs-blogger-analyzer`
> 请先 `git clone` 并 `npm install` 再使用本 Skill。

## 执行流程（Claude 按此步骤操作）

### Step 0: 检查环境
确认项目目录存在，且 `node_modules` 已安装。如果没有，引导用户先 `npm install`。

### Step 1: 获取 Cookie
引导用户从浏览器导出小红书 Cookie，保存为 `cookies.json`。

### Step 2: 采集数据
运行 `node scrape.js --url <博主URL> --cookies cookies.json`。等待采集完成。

### Step 3: 分析数据
运行 `python analyze_all.py --notes ./data/all_notes.json --output ./data/results.json`。

### Step 4: 生成报告
运行 `python generate_reports.py --input ./data/results.json --output ./reports`。

### Step 5: 展示结果
告诉用户报告已生成到 `./reports/` 目录，简要总结核心发现。

## 二、数据采集规范

### 2.1 所需数据

| 数据类型 | 采集方式 | 最低数量 | 理想数量 |
|---------|---------|---------|---------|
| 笔记元数据 | API: user_posted (翻页) | 200 | 全部 |
| 笔记正文 | 逐篇访问 explore/{note_id} | Top 20 + Bottom 20 | Top 50 + Bottom 50 + 随机 50 |
| 评论 | 从正文页面提取 | — | Top 10 笔记的前 10 条评论 |
| 博主信息 | API: user/me + SSR HTML | 1 | — |
| 封面图 | 从 API 或页面提取 | — | 全部 |

### 2.2 数据字段定义

```
Note {
  note_id: string          # 笔记唯一 ID
  xsec_token: string       # 安全令牌（访问单篇需要）
  title: string            # 标题
  type: 'normal' | 'video' # 内容类型
  liked_count: number      # 点赞数
  sticky: boolean          # 是否置顶
  created_at: string       # 发布时间（从正文提取）
  tags: string[]           # 标签列表
  content_text: string     # 正文全文
  comments: Comment[]      # 评论列表（可选）
  cover_url: string        # 封面图 URL（可选）
}

Comment {
  text: string
  liked_count: number
  reply_count: number
  timestamp: string
}
```

## 三、分析维度详解

### 维度 1：博主画像

#### 分析内容
- **基础信息**: 昵称、简介、IP、认证、粉丝量级
- **定位提炼**: 一句话概括博主定位，评估定位清晰度
- **人设要素**: 语言风格、视觉风格、人格化特征
- **目标受众**: 从内容推断核心受众画像

#### 分析方法
```
定位清晰度评分 = 简介与内容一致性 × 垂直度 × 人设鲜明度
垂直度 = 核心领域笔记数 / 总笔记数
```

#### 输出
- 博主画像卡片（120 字以内）
- 定位演变时间线（从简介变化和早期内容推断）

### 维度 2：内容结构分析

#### 2.1 主题分类体系

建立多级分类框架，对每篇笔记进行分类标注：

```
一级分类         二级分类                示例
AI模型/产品      大模型评测              DeepSeek V4 实测
                 AI工具推荐              CC Switch 体验
                 API/平台更新            OpenAI 新功能

教程/干货        保姆级教程              【保姆级】用 CC
                 Prompt 技巧             Prompt 合集分享
                 编程/开发               Codex 编程指南

观点/思考        行业趋势                AI 杀死破折号
                 社会影响                AI 与写作
                 个人感悟                做了三年AI

工具/产品发布    Skill 发布              「洁癖.Skill」
                 网站/平台               AIHOT 上线
                 开源项目                OpenClaw

热点追踪        产品发布追踪            Claude Opus 4.7 上线
                 行业新闻                Google 视频编辑

个人/日常        生活记录                感谢陪伴
                 趣事分享                发布会 Party

商业/活动        招聘                    AI博主招人
                 线下活动                AIFUT 大会
                 合作推广                品牌合作
```

#### 2.2 内容形态分析

| 分析项 | 方法 |
|-------|------|
| 图文 vs 视频占比 | 按 `type` 字段统计 |
| 图文/视频互动效率对比 | 计算类型均赞比 |
| 内容长度分布 | 正文文本字数统计 |
| 系列化内容识别 | 关键词聚类（"系列"、"第X篇"） |
| 内容日历规律 | 按时间维度归一化到周/月 |

#### 2.3 内容密度分析

```python
内容发布密度 = 笔记数 / 时间跨度(月)
各内容类型占比 = 类型笔记数 / 总笔记数
类型趋势 = 按月统计各类型笔记占比变化
```

#### 输出
- 主题分布饼图/柱状图
- 内容类型 vs 互动量的散点图
- 系列内容识别列表
- 内容密度时间线

### 维度 3：标题与文案分析

#### 3.1 标题公式提取

对每个标题进行结构拆解：

```
结构拆解模板:
  [情绪词/感叹] + [主语/产品名] + [动作/变化] + [效果/价值] + [标点符号]

示例:
  "我靠！DeepSeek多模态真的来了！？？"
  → 情绪词: 我靠 | 主语: DeepSeek | 动态: 来了 | 标点: ！？？

  "用了三年AI，我特别想说......"
  → 时间锚: 三年 | 领域: AI | 钩子: 想说...... | 省略号制造悬念

  "【保姆级】一文带你在手机玩CC！！"
  → 标签化: 【保姆级】 | 承诺: 一文带你 | 场景: 手机 | 目标: 玩CC
```

**标题变量提取**: 统计所有标题的结构组成要素出现频率

| 变量 | 定义 | 示例值 |
|------|------|--------|
| 情绪前缀 | 表达情绪的词语 | 我靠、麻了、绝了、疯了吧 |
| 时间锚 | 时间段 | 三年、几个月、凌晨四点 |
| 数字 | 具体/模糊数字 | 一文、三年、10个、60个小时 |
| 产品名 | 提及的产品 | DeepSeek, Claude, ChatGPT |
| 动词 | 核心动作 | 来了、上线、实测、公开 |
| 价值承诺 | 给读者的好处 | 总结合集、保姆级、一招 |
| 标点 | 情绪强度 | ！！？？、...、！！ |
| 括号 | 内容标签化 | 【完整版】【附网址】 |

#### 3.2 标题模式聚类

用聚类算法将所有标题分组，识别不同标题模式并统计各模式的互动表现：

```
模式 A: 情绪 + 产品 + 动态 + 感叹
  例: "我靠！DeepSeek多模态真的来了！？？"
  平均表现: X 赞
  代表笔记: [n1, n2, n3]

模式 B: 时间 + 领域 + 钩子
  例: "用了三年AI，我特别想说......"
  平均表现: X 赞

模式 C: 【标签】+ 价值承诺
  例: "【保姆级】一文带你在手机玩CC！！"
  平均表现: X 赞
```

#### 3.3 正文结构分析

对 Top 20 笔记的正文进行结构拆解：

```
段落结构模板:
  Hook (前 3 句) → 背景铺垫 → 核心内容 → 观点升华 → 互动引导

Hook 类型分类:
  - 反常识: "AI 不应该..."
  - 故事: "事情是这样的..."
  - 数据: "调研了 X 个..."
  - 提问: "你知道...?"
  - 共情: "说实话..."

结尾 CTA 分类:
  - 收藏引导: "建议先收藏"
  - 关注引导: "关注我，持续更新"
  - 评论引导: "你怎么看？"
  - 分享引导: "转发给需要的朋友"
```

#### 3.4 文案风格分析

| 维度 | 指标 | 分析方法 |
|------|------|---------|
| 口语化程度 | 语气词密度（呢、啊、吧、吗） | 统计每千字频率 |
| 情绪强度 | 感叹号密度 + 情绪词密度 | NLP 情感打分 |
| 信息密度 | 知识点数量 / 字数 | 人工标注样本 |
| 修辞手法 | 比喻、排比、反问使用 | 规则检测 |
| 互动密度 | 每段平均字数 | 段落数/总字数 |

#### 输出
- 标题公式库（含各公式互动表现排名）
- 最高效 Hook 类型 TOP 3
- 正文结构模板（可直接套用）
- 文案风格雷达图

### 维度 4：互动归因分析

#### 4.1 互动分布

```python
# 分布密度
互动集中度 = Top 10% 笔记的总赞 / 全部笔记总赞
# 帕累托分析
爆款贡献率 = 累计达到 80% 总赞的笔记占比
```

#### 4.2 爆款 vs 普通对比

对 Top 20 和 Bottom 20 进行系统性对比：

| 对比维度 | Top 20 特征 | Bottom 20 特征 | 差异显著性 |
|---------|------------|---------------|-----------|
| 标题长度 | — | — | t-test |
| 标题句式 | — | — | χ² test |
| 内容类型 | — | — | χ² test |
| 发布时间 | — | — | 分布检验 |
| 标签数量 | — | — | t-test |
| 正文长度 | — | — | t-test |
| 封面风格 | — | — | 视觉对比 |
| 是否系列化 | — | — | χ² test |

#### 4.3 互动预测模型（可选）

```python
# 多元线性回归 / 决策树
liked_count ~ title_length_has_exclamation_has_number
            + content_type + tag_count
            + has_bracket + has_emotional_word
            + content_length + ...
```

#### 4.4 评论情感分析

对 Top 10 笔记的高赞评论进行情感分类：

```
评论分类:
  - 提问/求助: "请问这个怎么用？"
  - 共鸣/认可: "说得太对了"
  - 讨论/补充: "我觉得还可以..."
  - 感谢/支持: "感谢分享！"
  - 纯收藏: "先马住"
```

统计各类型占比，分析粉丝的核心诉求。

#### 输出
- 互动分布曲线（帕累托图）
- 爆款 vs 普通对比矩阵
- 评论情感分类饼图
- 互动影响因素排序

### 维度 5：成长轨迹分析

#### 5.1 阶段划分（从数据中自动识别）

```
算法: 基于点赞趋势 + 内容类型变化 + 发布频率 做断点检测

阶段定义:
  冷启动期: 首条笔记 → 第一条爆款出现
  上升期: 第一条爆款 → 稳定月均 X 篇爆款
  爆发期: 快速涨粉阶段
  成熟期: 互动稳定 + 商业化
  瓶颈/转型: 互动下滑 / 内容方向调整
```

#### 5.2 关键指标曲线

| 曲线 | X 轴 | Y 轴 | 洞察点 |
|------|------|------|-------|
| 发布频率 | 时间 | 每周笔记数 | 勤奋度变化 |
| 互动趋势 | 时间 | 均赞/周 | 内容质量变化 |
| 爆款密度 | 时间 | 爆款数/月 | 成长拐点 |
| 内容类型迁移 | 时间 | 类型占比 | 策略调整 |
| 互动收敛性 | 时间 | 点赞标准差 | 稳定性变化 |

#### 5.3 转折点分析

对所有重大转折点进行归因：

```python
转折点检测条件:
  - 单篇比前 10 篇均赞高 3 倍 → 增长拐点
  - 连续 10 篇均赞下降 30% → 瓶颈期
  - 内容类型占比变化 > 20% → 策略转型
  - 发布频率变化 > 50% → 投入度变化
```

每个转折点分析：
- 触发原因（那篇笔记是什么？）
- 转变方向（内容/形式/风格变化）
- 效果（互动变化 + 持续时间）

#### 输出
- 成长阶段时间线（带关键事件标注）
- 互动趋势折线图 + 移动平均线
- 转折点归因分析表
- 成长曲线 vs 同赛道平均对比

### 维度 7：商业化分析

#### 7.1 商业化类型识别

```python
商业化信号检测:
  - 标题中出现品牌名非 AI 产品 → 推广
  - 内容为纯产品体验 → 可能是商单
  - @品牌官方账号 → 合作
  - 招聘/活动信息 → 团队化运营
  - 自有产品/工具发布 → 变现产品
```

#### 7.2 商业化指标

| 指标 | 定义 |
|------|------|
| 商业化率 | 商业笔记数 / 总笔记数 |
| 商业内容均赞 | 商业笔记平均互动 |
| 自然内容均赞 | 非商业笔记平均互动 |
| 商业 vs 自然差距 | 商业内容相较自然内容的表现差异 |
| 商业化周期 | 从第一篇文章到第一次商业合作的时间 |

#### 7.3 变现模型推断

```python
变现路径识别:
  - 接广（品牌合作）
  - 自有工具/产品（Skills、AIHOT）
  - 付费社群/知识星球
  - 咨询/培训
  - 线下活动（AIFUT 大会）
  
推断依据:
  - 内容中提及的收费产品/服务
  - 招聘信息（团队规模推断）
  - 活动信息
  - 合作品牌层级
```

#### 输出
- 商业化时间线
- 变现模型全景图
- 商业 vs 自然内容表现对比
- 品牌合作层级评估

## 五、分析质量标准

### 5.1 每个结论必须有数据支撑

```
✅ "教程类内容均赞 1,555，高于整体均值 489 的 3.2 倍"
❌ "教程类内容表现最好"
```

### 5.2 对比基准

- 与博主自身均值对比
- 与同赛道均值对比（需横向采集）
- 与平台大盘均值对比（公开数据）

### 5.3 可信度标注

```
[Confirmed]     → 数据直接支撑（样本量 > 30）
[Indicative]    → 数据趋势性指向（样本量 10-30）
[Speculative]   → 基于有限样本的推断（样本量 < 10）
[Need More Data] → 缺乏足够数据验证
```

### 5.4 报告质量检查清单

- [ ] 每个数据结论标注了样本量
- [ ] 每个建议有数据支撑
- [ ] 明确区分了"已验证"和"推测"
- [ ] 提供了可直接使用的模板
- [ ] 报告可在 5 分钟内读完重点
- [ ] 生成了至少 1 张可视化图表
- [ ] 提出了可验证的假设
